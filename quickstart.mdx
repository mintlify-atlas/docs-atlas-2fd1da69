## Get started with sandboxec

This quickstart guide will help you install sandboxec and run your first sandboxed command.

<Steps>
  <Step title="Check kernel compatibility">
    sandboxec requires Linux kernel 5.13+ with Landlock support enabled.
    
    Check your kernel version:
    
    ```bash
    uname -r
    ```
    
    <Note>
    Network restrictions (TCP bind/connect) require kernel 6.7+ (Landlock ABI v4+)
    </Note>
  </Step>

  <Step title="Install sandboxec">
    Choose your preferred installation method:
    
    <Tabs>
      <Tab title="Install Script">
        ```bash
        curl -sSL https://get.sandbox.ec | sh
        ```
      </Tab>
      
      <Tab title="Go Install">
        ```bash
        go install go.sandbox.ec/sandboxec@v0.3.0
        ```
        
        <Note>Requires Go 1.24.0 or later</Note>
      </Tab>
      
      <Tab title="Pre-built Binary">
        Download from the [releases page](https://github.com/sandboxec/sandboxec/releases/latest)
      </Tab>
    </Tabs>
  </Step>

  <Step title="Run your first sandboxed command">
    Sandbox a simple command with filesystem restrictions:
    
    ```bash
    sandboxec --fs rx:/usr -- /usr/bin/echo "Hello from sandbox!"
    ```
    
    This command:
    - Grants read+execute (`rx`) access to `/usr`
    - Runs `/usr/bin/echo` with the argument "Hello from sandbox!"
    - Denies access to everything else on the filesystem
    
    <Info>
    The `--` separator is optional but recommended to clearly separate sandboxec options from the wrapped command and its arguments.
    </Info>
  </Step>

  <Step title="Try filesystem isolation">
    See how sandboxec blocks unauthorized filesystem access:
    
    ```bash
    # This works - /usr is allowed
    sandboxec --fs rx:/usr -- /usr/bin/ls /usr
    
    # This fails - /home is not in the allow-list
    sandboxec --fs rx:/usr -- /usr/bin/ls /home
    ```
    
    The second command will fail with a permission denied error because `/home` was not granted access.
  </Step>

  <Step title="Add network restrictions">
    Sandbox a command that needs network access:
    
    ```bash
    sandboxec \
      --fs rx:/usr \
      --net c:443 \
      -- /usr/bin/curl https://example.com
    ```
    
    This allows:
    - Read+execute access to `/usr` for running curl and its dependencies
    - TCP connect (`c`) on port 443 for HTTPS
    
    <Warning>
    Network restrictions require Linux kernel 6.7+ (Landlock ABI v4+). On older kernels, use `--best-effort` to ignore unsupported features.
    </Warning>
  </Step>

  <Step title="Use multiple filesystem rules">
    Complex commands often need access to multiple paths:
    
    ```bash
    sandboxec \
      --fs rx:/usr \
      --fs rx:/bin \
      --fs rw:/tmp \
      --fs rw:$PWD \
      -- your-build-command
    ```
    
    This grants:
    - Read+execute to `/usr` and `/bin` for system binaries
    - Read+write to `/tmp` for temporary files
    - Read+write to current working directory (`$PWD`) for build output
  </Step>
</Steps>

## Common Patterns

### Sandbox third-party CLI tools

```bash
sandboxec \
  --fs rx:/usr \
  --fs rw:$PWD \
  --net c:443 \
  -- npm install
```

### Test untrusted scripts

```bash
sandboxec \
  --fs rx:/bin \
  --fs rx:/usr \
  --fs rw:/tmp \
  -- bash ./untrusted-script.sh
```

### AI agent command execution

```bash
sandboxec \
  --fs rx:/usr \
  --fs rw:/tmp \
  --fs rw:/path/to/workspace \
  --net c:443 \
  -- agent-generated-command
```

## Troubleshooting

<Accordion title="Command fails with 'permission denied'">
  The command needs access to additional filesystem paths. Common missing dependencies:
  
  - System libraries: Add `--fs rx:/lib` and `--fs rx:/usr/lib`
  - Shared objects: Add `--fs rx:/lib64`
  - Configuration files: Add `--fs r:/etc`
  
  Or use `--unsafe-host-runtime` to automatically allow runtime dependencies (with security trade-offs).
</Accordion>

<Accordion title="Network restrictions don't work">
  Network restrictions require Linux kernel 6.7+ (Landlock ABI v4+).
  
  Check your kernel version:
  ```bash
  uname -r
  ```
  
  If your kernel is older, use `--best-effort` to run without network restrictions:
  ```bash
  sandboxec --best-effort --fs rx:/usr --net c:443 -- curl https://example.com
  ```
</Accordion>

<Accordion title="'Landlock not supported' error">
  Your kernel was built without Landlock support or is older than 5.13.
  
  Solutions:
  - Update to a newer kernel (5.13+)
  - Check if Landlock is enabled: `cat /sys/kernel/security/lsm`
  - Use containers/VMs instead for older systems
</Accordion>
